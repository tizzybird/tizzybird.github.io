MapReduce

MapReduce 是一套由 Google 工程師開發的 library。他們發現大部分的資料運算大概都由兩個部份組成：

  1. Map: 將輸入資料對應到鍵值對 (key-value pair)
  2. Reduce: 將輸入資料轉換成最終輸出
    以下先介紹整個架構，在介紹 Mapper 和 Reducer 的功能與特性。

架構
MapReduce 的架構可以分為三個部份：

  1. User Program: 使用者自訂的程式，負責將 input 資料切成 16 ~ 64MB 大小的片段，作為 mapper 的輸入。此外也需定義 mapper 跟 reducer 所要執行的 function (User Defined Function, UDF)。之後把執行程式複製多份，分別執行 Master 與 Workers。
  2. Master: 負責監控 Worker 的狀態並指派工作給閒置的 Worker，再從 Worker 取得執行結果。
  3. Workers (Slaves): 負責執行 Map / Reduce 的運算，根據分配到的工作可以是 Mapper 或 Reducer:
    • Mapper: 將輸入資料轉成鍵值對暫存到記憶體，之後再寫入到 local disk。完成工作後會通知 Master 資料所在的位置，Master 會再把這些位置告訴 Reducer 用 RPC 的方式讀取作為其輸入。
    • Reducer: 從 Master 獲得中間鍵值對 (intermediate key-value pairs) 所在的位置後，再用 RPC 的方式讀取。

1. Fault Tolerance 容錯

錯誤分類

根據錯誤的層級以及影響，由小至大，有以下幾種分類：

  1. Fault
  2. Error
  3. Failure

其中以 Fault 影響的層面最小，Failure 的影響層面最大，因此可以大概寫成下面的傳遞關係：
Fault -> Error -> … -> Error -> Failure

這邊要注意的是一個 Error 不必然會造成 Failure；Failure 可以是因為一個或多個 Error 引起的。

不過一般情形下，Fault/Failure 通常是相互代稱。

Failure 種類

  1. Silent/Crash: 元件停止運作，這種情形相對容易 debug
  2. Non-silent: 元件產生錯誤的輸出結果，這種情形要避免讓錯誤的結果傳遞下去影響到其他的元件
  3. Malicious: adversarial behavior (non-silent)，拜占庭將軍問題，如何抓出叛徒

Fault 產生的原因

  • 由 Hardware 造成
    • RAM: bit flip 問題 -> 用 Parity bit 或 ECC 解決
    • Disk/SSD: 讀寫資料毀損 -> checksum 或 RAID
    • CPU
  • 由 Software 造成
    • Deterministic bugs (Bohrbugs): 給特定的條件就會產生
    • Non-deterministic bugs (Heisenbugs): 給特定的條件也不一定會產生，難以複製
    • Configuration faults

容錯機制

想像若一台沒有 fault tolerance 的 server 每個月都會故障一次，則一個有 10 台相同 server 的機房，每 3 天就會有一台機器故障；若有 100 台，則每 7 小時就會有一次故障；有 1000 台，則每 50 分鐘就會故障！
因此我們需要有容錯的機制來處理各種大大小小的 fault 跟 error，避免他們變成 failure 造成服務終止。
Fault tolerance 依賴 Fault assumption，我覺得有點像是情境分析，就是把所有各種可能會造成錯誤的情況都列出來，然後再設計相對應的方法處理。Fault assumption 有所謂的覆蓋率，也就是 assumption 是否能 cover 到所有可能發生錯誤的情形。這個覆蓋率不可能為 100%，畢竟錯誤任何地方都可能會發生。

可靠性評估

我們可以用以下方法來評估一個系統的可靠度：

  • Reliability: 系統上線的時間
  • Maintainability: 系統離線的時間
  • Availability: 上線時間 / 離線時間，也就是 Reliability / Maintainability

如果不幸發生 Failure 要怎麼避免影響到整台 Server?

如果系統不幸發生 Failure，但我們又不希望服務因此終止，要如何讓服務能在此情況繼續運作?
Ans: 使用分散式系統，將提供相同功能的 program 複製到多個 server 執行，如此就算其中有幾台發生 Failure，服務也不會因此終止。此外還可以提升服務的 Throughput。不過需要注意 concurrency 的問題。

leetcode-124

Posted on 2020-05-23 | In Leetcodes |

Description

Question: Given a non-empty binary tree, find the maximum path sum.

For this problem, a path is defined as any sequence of nodes from some starting node to any node in the tree along the parent-child connections. The path must contain at least one node and does not need to go through the root.

Leetcode link: 124. Binary Tree Maximum Path Sum

Idea

Since the maximum path sum should be returned, we have to loop over every node at least once. A variable for recording the maximum should created.

A maximum path consists of many maximal paths in local areas. That is, if a node parent is on a maximum path, then:

  • Its left subtree must return a maximum path, or
  • Its right subtree must return a maximum path, or
  • Neither left subtree nor right subtree return a maximum path except itself (the case happens when the sum of path is smaller than 0)

As a result, the problem could be splitted into smaller problems. At each recursion, the path sums of left subtree and right subtree are compared. The recursion stops when a node is null. The maximum path is updated if current path sum is larger. Finally, the maximal sum of current tree is returned.

Test cases

Input Expected Output
nums=[1,2,3] 6
nums=[-10,9,20,null,null,15,7] 42

Description

Question: Suppose an array sorted in ascending order is rotated at some pivot unknown to you beforehand.

(i.e., [0,1,2,4,5,6,7] might become [4,5,6,7,0,1,2]).

You are given a target value to search. If found in the array return its index, otherwise return -1.

You may assume no duplicate exists in the array.

Your algorithm’s runtime complexity must be in the order of O(log n).

Leetcode link: 33. Search in Rotated Sorted Array

Idea

If we list all possible rotation of an array, we can notice below attributes:

  • If the middle value is smaller than the rightmost element, than the sequence from the middle to the rightmost is ascending.
  • Otherwise, the sequence from the leftmost to the middle is ascending.

Based on above attributes, we can further know if the target is right or left to the middle. If the target is larger than the middle but smaller than the rightmost value, the left bound should be shrinked. Otherwise, the right bound should be moved to left.

<413> Arithmetic Slices

Posted on 2020-02-18 | In Leetcodes |

Description

Question: A sequence of number is called arithmetic if it consists of at least three elements and if the difference between any two consecutive elements is the same.

A zero-indexed array A consisting of N numbers is given. A slice of that array is any pair of integers (P, Q) such that 0 <= P < Q < N.

A slice (P, Q) of array A is called arithmetic if the sequence:
A[P], A[p + 1], …, A[Q - 1], A[Q] is arithmetic. In particular, this means that P + 1 < Q.

The function should return the number of arithmetic slices in the array A.

Leetcode link: 413. Arithmetic Slices

Quick Sum-up:

  • Input: an array
  • Output: a number indicating valid arithmetic slices

Idea

We can observe that the result of an array with k elements depends on whether the difference of k^th and (k-1)^th element is the same as the difference of its previous k-1 elements. If they are same, the number of previous result should be considered again in the join with k^th element, plus a newly generated (k-2, k-1, k) slice. Otherwise, we don’t have to consider previous results. That’s what variable “dp” does. => Pass, 44ms

<221> Maximal Square

Posted on 2020-02-03 | In Leetcodes |

Description

Question: Given a 2D binary matrix filled with 0’s and 1’s, find the largest square containing only 1’s and return its area.

Leetcode link: 221. Maximal Square

Quick Sum-up:

  • Input: a 2D array
  • Output: a number indicating the number of 1 in the largest square

Idea

For each cell in the matrix, if its value is 1, try expanding a square from top left to down right until a 0 is encountered. A 2D array is used for recording the maximal length of the expanded square at each cell. The speed of expanding square can be increased by using the 2D array record, since the initial length used for expanding a square is the maximum of records of top cell and left cell minus 1. => Pass, 3ms

<139> Word Break

Posted on 2020-01-26 | In Leetcodes |

Description

Question: Given a non-empty string s and a dictionary wordDict containing a list of non-empty words, determine if s can be segmented into a space-separated sequence of one or more dictionary words.

Note: The same word in the dictionary may be reused multiple times in the segmentation.
You may assume the dictionary does not contain duplicate words.

Leetcode link: 139. Word Break

Quick Sum-up:

  • Input: a string and an array of words
  • Output: a boolean value indicating whether the string can be assembled by the words
  • The string is non-empty.
  • The array is non-empty.

Idea

  1. Compare the string with words in the dictionary. If a word appears in the string, slice the string and put the substring to the next round of matching until the string is empty. => TLE, fail on the 4th case
  2. Use BFS, the queue of BFS stores the start index of next round. In addition, we need to record whether an index has been stored in the queue. => Pass, 68ms
